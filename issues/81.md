## Summary

Add end-to-end / integration tests for the backend API that exercise full request flows against a running Hono server, complementing the existing unit tests that use mocked Kubernetes clients.

## Motivation

Backend routes currently only have unit tests with mocked dependencies. E2e tests against a live server would catch issues in middleware chaining, error handling, serialization, and cross-route interactions that unit tests miss.

## Scope

### Routes missing test coverage entirely
- [x] `/api/autoscaler` — detection and status endpoints
- [x] `/api/aiconfigurator` — status, analyze, and normalize-gpu endpoints

### Routes with partial coverage (missing sub-endpoints)
- [x] `/api/models/search` — search endpoint
- [x] `/api/models/:id/gguf-files` — GGUF files endpoint
- [x] `/api/deployments/:name/manifest` — manifest endpoint
- [x] `/api/deployments/:name/pending-reasons` — pending reasons endpoint
- [x] `/api/deployments/:name/logs` — logs endpoint
- [x] `/api/installation/providers/*` — various provider installation endpoints

### Multi-step flow tests
